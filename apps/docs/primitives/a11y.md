# A11y

The A11y primitive provides utilities for handling platform-specific accessibility differences, particularly between Web (ARIA) and React Native (Accessibility props).

## Installation

```bash
npx shadniwind@latest add a11y
```

## Usage

### nativeRole

Maps a web aria-role to a React Native `accessibilityRole`.

```tsx
import { nativeRole } from "~/components/primitives/a11y";
import { View } from "react-native";

<View accessibilityRole={nativeRole("button")} />;
```

### ariaAttr

Standardizes boolean accessibility states.

```tsx
import { ariaAttr } from "~/components/primitives/a11y";
import { View } from "react-native";

<View aria-checked={ariaAttr(isChecked)} />;
```

## API Reference

### nativeRole

```ts
function nativeRole(role: string): AccessibilityRole
```

### ariaAttr

```ts
function ariaAttr(condition: boolean | undefined): boolean | undefined
```
Returns `true` if condition is truthy, otherwise `undefined`.
